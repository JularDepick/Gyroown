# Gyroown GUI 搭建计划 v0.1.1

> 目标：搭建完整 GUI 骨架，预留核心功能接口（加密/存储算法不实现）。
> 分 3 期，按逻辑依赖推进，无时间约束。

---

## 0. 技术审查修正（v0.1.1 更新）

以下为审查发现的 API/架构问题及修正，已融入各期细节。

| 问题 | 严重度 | 修正 |
|------|--------|------|
| WinUI 3 无内置 `NotifyIcon` | 🔴 | 使用 **H.NotifyIcon** NuGet 包 |
| 单实例检测未指定机制 | 🔴 | 使用 **`AppInstance.FindOrRegisterForKey()`**（WindowsAppSDK） |
| `~/.Gyroown` 为 Unix 路径 | 🔴 | Windows 上使用 **`%USERPROFILE%\.Gyroown`** |
| 标题栏拖拽区域未设置 | 🔴 | `ExtendsContentIntoTitleBar` + `Window.SetTitleBar(dragElement)` |
| 最小/大/关闭按钮需自绘 | 🔴 | 自定义按钮 + P/Invoke `ShowWindow` / `SendMessage` |
| `StorageItems` 延迟渲染实现复杂 | 🟡 | 拖出时预创建临时文件到 `Path.GetTempPath()`，`StorageItems` 直接引用，拖放完成后安全擦除 |
| `TreeView` 需手动构造 `TreeViewNode` | 🟡 | 不直接绑定 `ItemsSource`，改为代码构建节点树 |
| 密码控件选型 | 🟡 | PIN 改用 6 个 `PasswordBox`（单字符模式），`PasswordBox.PasswordRevealMode` 处理显隐 |
| 大文件加解密无进度 | 🟡 | 接口预留 `IProgress<double>` 参数 |
| `IVaultService.Initialize(password)` 语义错误 | 🟡 | 改为 `Initialize(derivedKey)` — 不传原始密码 |
| MicaBackdrop Win10 兼容 | 🟡 | 运行时检查 `MicaBackdrop.IsSupported`，降级为 `SolidColorBrush` |
| 安全擦除流程不完整 | 🔵 | 先覆写随机数据 → Flush → 再 DeleteOnClose |
| 缺少错误处理策略 | 🟡 | 所有桩方法包裹 try-catch，UI 层通过 `ContentDialog` 展示异常 |
| 缺少自动锁定 | 🔵 | `IPasswordService` 预留 `AutoLockTimeout` 属性，通过 `DispatcherTimer` 空闲检测 |
| 图片密码缩放映射 | 🟡 | 图片以 `Uniform` 模式显示，点击坐标按实际渲染区域反算比例 |

## 第 1 期：应用外壳 & 服务接口定义

### 1.1 应用生命周期

- 后台驻留：主窗口 `Closed` 事件中 `e.Handled = true; this.Hide();`，进程不退出。
- 托盘：**H.NotifyIcon**（`H.NotifyIcon.WinUI` NuGet）。右键菜单：「打开 Gyroown」「退出」。
  左键单击托盘 → 恢复窗口。
- 单实例：`AppInstance.FindOrRegisterForKey("Gyroown.SingleInstance")`，
  若已有实例 → `AppInstance.RedirectActivationTo()` 激活已有窗口 → 当前进程退出。
- 启动流程：`App` 构造函数中检查 `IPasswordService.IsPasswordSet` → 未设置显示 `PasswordSetupWindow`，已设置显示 `UnlockWindow` → 解锁后显示 `MainWindow`。

### 1.2 主窗口骨架（MainWindow）

布局模拟 Windows 文件资源管理器：

```
┌──────────────────────────────────────────────┐
│ TitleBar (自定义标题栏 + 主题色切换按钮)       │
├──────────────────────────────────────────────┤
│ Ribbon / CommandBar (文件操作工具栏)           │
├──────────┬───────────────────────────────────┤
│ Sidebar  │ Content Area                      │
│ (导航树) │  - 视图切换 (Details/Icons/Tiles)  │
│          │  - 文件列表 (ListView/GridView)    │
│          │  - 空状态占位                      │
├──────────┴───────────────────────────────────┤
│ StatusBar (选中项数、加密状态)                  │
└──────────────────────────────────────────────┘
```

**具体控件**：
- `TitleBar`：`ExtendsContentIntoTitleBar = true`，`Window.SetTitleBar(titleBarGrid)` 设定拖拽区域。
  左侧应用图标 + "Gyroown"，中间搜索框（桩），右侧主题切换下拉 + 自定义最小化/最大化/关闭按钮。
  关闭按钮 → 隐藏窗口（`Hide()`），不退出。最小/大化通过 P/Invoke `ShowWindow(SW_MINIMIZE/SW_MAXIMIZE)`。
- `CommandBar`：`NewFolder`, `Import`, `Export`, `Delete`, `Properties` 按钮。
- `Sidebar`：`TreeView`，代码构建 `TreeViewNode` 节点树（桩数据）。
- `Content Area`：视图切换栏 + 内容区，根据视图模式切换 `ListView`(Details) / `GridView`(Icons/Tiles)。
- `StatusBar`：底部 `Grid`，选中计数 + 仓库路径 + 加密锁图标。
- `MicaBackdrop`：运行时 `if (MicaBackdrop.IsSupported)` 否则降级纯色背景。

### 1.3 密钥架构（关键设计）

```
┌─────────────────────────────────────────────────┐
│ 用户密码 (PIN/手势/自定义/图片)                     │
│   │  └── 仅用于身份认证 + 保护内部密钥              │
│   ▼                                              │
│ IPasswordService.ValidateAsync(credential)        │
│   │  └── 通过后，用户密码派生 userKey               │
│   ▼                                              │
│ IEncryptionService.DeriveUserKey(password, salt)  │
│   │  └── 用 userKey 解密 vault-key.enc            │
│   ▼                                              │
│ 内部固定密钥对 (非对称，首次生成后原始明文永不改变)    │
│   ├── 私钥 (privateKey)  → 用于加密文件（写入保密柜）  │
│   └── 公钥 (publicKey)   → 用于解密文件（从保密柜取出） │
│   二者均不可公开，启动时解密加载到内存，进程死后消失       │
│   ▼                                              │
│ EncryptFileAsync(stream, privateKey)  加密写入     │
│ DecryptFileAsync(stream, publicKey)   解密读取      │
│   └── 密钥对原始明文是唯一加解密文件的凭证              │
│                                                  │
│ 落盘: %USERPROFILE%\.Gyroown\                    │
│   ├── auth\vault-key.enc (密钥对密文,              │
│   │     由 userKey 双向加密储存，改密码仅重加密此文件) │
│   ├── key\portrait.*     (图片密码)                │
│   └── data\              (加密文件)               │
└─────────────────────────────────────────────────┘
```

**关键约束**：
- `GenerateVaultKeyPair()` 仅在首次设置时调用 **一次**，返回 `(privateKey, publicKey)`，之后绝不重生成。
- 用户改密码 → `EncryptVaultKeyPair(keyPair, newUserKey)` 覆写 `auth\vault-key.enc`，密钥对明文不变，文件不动。
- 卸载时只有用户明确选择「删除数据及密钥」才能删除 `auth\` 和 `data\`。

### 1.4 服务接口定义

所有接口放在 `Services/`，实现类均为 `throw NotImplementedException` 桩。

| 接口 | 职责 | 关键方法 |
|------|------|----------|
| `IPasswordService` | 用户身份认证与会话 | `ValidateAsync(credential) → (bool ok, byte[] userKey)`, `SetupAsync(credential)`, `IsPasswordSet`, `ChangePassword(oldCred, newCred)` — 内部重加密 vault-key.enc, `Lock()`, `IsLocked`, `AutoLockTimeout`, 事件 `Unlocked`, `Locked` |
| `IEncryptionService` | 两层密钥 + 文件加解密（非对称密钥对） | `DeriveUserKey(password, salt) → byte[]`, `GenerateVaultKeyPair() → (byte[] privateKey, byte[] publicKey)`, `EncryptVaultKeyPair(keyPair, userKey) → byte[]`, `DecryptVaultKeyPair(encrypted, userKey) → (byte[] privateKey, byte[] publicKey)`, `EncryptFileAsync(inStream, outStream, privateKey, progress)`, `DecryptFileAsync(inStream, outStream, publicKey, progress)` |
| `IVaultService` | 加密仓库 CRUD（持有密钥对引用，永不改变） | `ListItems()`, `ImportItemAsync(stream, progress)`, `ExportItemAsync(id, stream, progress)`, `DeleteItem(id)`, `MoveItem(id, newPath)`, `IsInitialized`, `Initialize(privateKey, publicKey)`, `VaultPath` |
| `IThemeService` | 主题色管理 | `CurrentTheme`, `SetTheme(theme)`, `GetAvailableThemes()`, 事件 `ThemeChanged` |
| `IDragDropService` | 拖放协调 | `HandleDropInAsync(files, progress)`, `HandleDragOutAsync(items, progress)` — 协调加解密与 UI 进度 |

### 1.6 通用设计决策

- **路径**：所有本地数据写入 `%USERPROFILE%\.Gyroown\`，结构：`auth\`（内部密钥对密文）、`data\`（加密文件）、`key\`（图片密码）、`settings.json`（配置）。
- **异常处理**：每个桩方法包裹 `try-catch`。UI 层拦截异常 → `ContentDialog` 展示用户友好消息。
- **后台锁定**：`IPasswordService` 提供 `DispatcherTimer` 空闲检测，`AutoLockTimeout` 可配（默认 0 = 不自动锁定，桩）。
- **密码控件复用**：四种密码控件各自实现 `IPasswordControl` 接口（`GetCredential()`, `Clear()`, `event Validated`），便于 `UnlockWindow` 和 `PasswordSetupWindow` 统一调用。

## 第 2 期：密码解锁系统

### 2.1 解锁窗口（UnlockWindow）

首次启动或锁定后显示，主窗口不可见。解锁成功 → 隐藏自身，显示主窗口。

### 2.2 四种密码类型

#### A. 6 位数字 PIN
- `NumberBox` / 6 个 `TextBox`（逐位输入自动跳转）。
- 只接受数字 0-9，输入满 6 位自动触发验证。

#### B. 九宫格手势图案
- 3×3 `Grid` 的 `Ellipse`（圆点），支持鼠标/触摸拖拽连线。
- 用 `Canvas` 覆层绘制连线（`Polyline`）。
- 记录访问序列（编号 0-8），最少连接 4 个点，不可重复。
- 设置时需绘制两次确认。

#### C. 自定义密码（6-32 位）
- `PasswordBox` + 显示/隐藏切换按钮。
- 字符集：`[a-zA-Z0-9]` + 可见标点 `!@#$%^&*()-_=+[]{};:'",.<>?/|~`。
- 设置时需输入两次确认。实时校验长度。

#### D. 图片密码
- 用户选择图片 → 复制到 `%USERPROFILE%\.Gyroown\key\portrait.*`。
- 图片以 `Uniform` 模式显示（保持宽高比，居中），点击坐标按实际渲染区域反算为 `(x_ratio, y_ratio)` 相对比例。
- 设置：用户依次点击 >=3 个坐标点，每个已点击点显示带序号的圆形标记（`Ellipse` + `TextBlock` 序号）。
- 允许「重置」按钮清除所有点重来。
- 验证：加载同一图片，用户按序点击。每次检查欧氏距离 ≤ 半径阈值（默认图片短边 5%，开发者可调）。
  - 正确：显示绿色序号圆 → 自动推进下一点。
  - 错误：全部点变红闪烁 500ms → 重置序列。
- 设置图片密码前需先设置一个备用密码（上述 A/B/C 之一），备用密码用于图片密码失效时恢复。

### 2.3 密码策略

| 类型 | 标识符 | 最小长度 | 错误锁定 |
|------|--------|----------|----------|
| PIN | `pin` | 6 | 连续 5 次错误锁定 30 秒 |
| Gesture | `gesture` | 4 点 | 连续 5 次错误锁定 30 秒 |
| Custom | `custom` | 6 字符 | 连续 5 次错误锁定 30 秒 |
| Picture | `picture` | 3 点 | 连续 5 次错误锁定 30 秒 |

锁定期间倒计时显示，不可尝试。

### 2.4 首次设置流程

```
首次启动 → 检查 %USERPROFILE%\.Gyroown\ 不存在则创建
         → 显示 PasswordSetupWindow
         → 4 个 RadioButton 选择密码类型
         → 对应设置控件收集凭据
         → 确认 →
           ① IEncryptionService.GenerateVaultKeyPair() 生成非对称密钥对
           ② IEncryptionService.DeriveUserKey(password, salt) 从用户密码派生 userKey
           ③ IEncryptionService.EncryptVaultKeyPair(keyPair, userKey) → 写入 auth\vault-key.enc
           ④ IPasswordService.SetupAsync(credential) 储存密码验证信息
           ⑤ IVaultService.Initialize(privateKey, publicKey) 加载密钥对到内存
         → 关闭设置窗口 → 显示 MainWindow
```

## 第 3 期：文件资源管理器式保密柜 UI

### 3.1 多视图文件列表

`VaultFileListView` 自定义控件，支持三种视图（用户可切换）：

| 视图 | 控件 | 显示内容 |
|------|------|----------|
| Details | `ListView` | 名称、加密后大小、原大小、类型、修改日期、加密状态 icon |
| Icons | `GridView` | 大图标 + 文件名 |
| Tiles | `GridView` | 中图标 + 文件名 + 类型 + 大小 |

文件项数据模型 `VaultFileItem`：
```csharp
class VaultFileItem {
    string Id;
    string Name;            // 加密存储的文件名
    string VirtualPath;     // 虚拟目录路径（如 "/documents/report.pdf"）
    long EncryptedSize;     // 加密后大小
    long OriginalSize;      // 原始大小
    string ContentType;
    DateTime CreatedAt;
    DateTime ModifiedAt;
    bool IsFolder;          // 虚拟文件夹
}
```

### 3.2 拖放（Drag & Drop）

#### 拖入（加密储存）
- 列表区域注册 `DragOver` + `Drop` 事件，接受 `DataPackageView` 中的 `StorageItems`。
- `Drop` → `IDragDropService.HandleDropInAsync(files, progress)`。
- 进度覆盖层：`ContentDialog` 或内嵌 `ProgressBar` + `ProgressRing`，
  「正在加密 {filename} ({N}/{Total})…」带取消按钮（`CancellationToken`）。
- 完成后自动刷新文件列表。

#### 拖出（解密拿取）
- 从列表拖动文件到外部 → `IDragDropService.HandleDragOutAsync(items, progress)`。
- 策略：在 `DragStarting` 时调用 `IVaultService.ExportItemAsync()` 解密到 `Path.GetTempPath()`，
  将临时文件作为 `StorageItems` 附加到 `DataPackage`。
  拖放完成后 **安全擦除**：打开文件 → 覆写随机数据 → `Flush` → 关闭 → `File.Delete()`。
- 进度覆盖层显示解密进度。

### 3.3 虚拟目录树

- `Sidebar` 使用 `TreeView`，绑定 `IVaultService.GetFolderTree()`（桩返回空）。
- 支持右键菜单：新建文件夹、重命名、删除。
- 文件夹为虚拟概念——加密存储中所有文件平铺，路径仅元数据。

### 3.4 右键上下文菜单

列表项右键菜单（`MenuFlyout`）：
- Open / Export / Rename / Delete / Properties

### 3.5 主题色切换

- `IThemeService` 管理颜色方案。
- TitleBar 右侧下拉按钮切换：跟随系统 / 浅色 / 深色 + 强调色选择器。
- 切换时更新 `MainWindow.Content.RequestedTheme` (WinUI 3 中主题按 `FrameworkElement` 设置) + 自定义 `ResourceDictionary` 刷色。
- 当前主题持久化到 `%USERPROFILE%\.Gyroown\settings.json`（桩）。

### 3.6 自定义标题栏

- `ExtendsContentIntoTitleBar = true` + `Window.SetTitleBar(titleBarGrid)` 设定拖拽区域。
- 左侧：应用图标 + "Gyroown"。
- 中间：搜索框（桩）。
- 右侧：主题切换 + 自定义最小化(`SW_MINIMIZE`) / 最大化(`SW_MAXIMIZE`) / 关闭按钮。
  按钮通过 P/Invoke `ShowWindow(hwnd, nCmdShow)` 实现。
  `hwnd` 通过 `WindowNative.GetWindowHandle(this)` 获取。
- 关闭按钮 → `this.Hide()`（不退出进程）。

### 3.7 状态栏

- 左侧：「{N} items」选中计数。
- 中间：仓库路径文本。
- 右侧：加密锁图标（🔒）+ "Encrypted" 文字。
