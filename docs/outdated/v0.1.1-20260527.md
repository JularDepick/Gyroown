## 执行策略

1. **任务拆分**：每个任务拆到可在单次工作周期内完成的粒度
2. **验收前置**：每个任务明确验收标准，完成后立即验收
3. **构建检查**：每次提交后测试工程师验证构建 0 错误
4. **文档同步**：功能变更后及时更新 DEVELOP.md
5. **不碰 git**：代码改动由开发直接编辑，不通过 git 操作
